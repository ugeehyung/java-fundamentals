- Git
- .git폴더가 있는 폴더의 내용을 모두 관리해주는 프로그램
- 조금이라도 변하면 모든 것을 알아채고 관리함
- 원래는 Git을 설치해야 하지만, Eclipse에 Git이 내장되어 있음

- GitHub
- 자기 컴퓨터에서 작업하던 .git으로 관리하던 폴더의 내용을 온라인에 올려서 공유
- GitHub로 저장하는 것을 push라고 하고, GitHub에서 자기 컴퓨터로 내려받는 것을 pull이라고 함
- Clone이라는 기능을 통해 모두 복사해서 가져올 수도 있음

- Git을 통한 폴더 관리의 기본
- git 폴더 내부에 무엇인가 변하면 git이 그것을 알고있음
- unstaged 상태: 이번 커밋에 포함하지 않을 변화들
- staged 상태: 이번 커밋에 포함시킬 변화들
- 해당 파일을 관리 상태로 두는 것을 add라고 함 
- 원래 콘솔화면에서는 git add 명령어를 이용해야 함
- commit: 현재 상태의 스냅샷을 찍음

- Commit
- 현재 stage에 추가된 파일들을 하나의 커밋으로 만든다
- Git은 언제든 관리하는 폴더의 모든 내용을 원하는 커밋으로 바꿀 수 있음

- .gitignore
- Git이 특정 확장자를 무시하도록 설정할 수 있음
- .gitignore 파일 내부에 작성한 확장자 또는 경로를 무시하게 됨 

- 이전 커밋들로 돌아가기
- 원래는 명령어를 써야함
- 우클릭 -> switch to -> commit 선택 후 원하는 commit을 선택
- show view -> Git Reflog 에서 원하는 커밋을 check out

- Branch
- 나뭇가지
- 하나의 폴더를 다양한 상태로 나누는 것
- git repo -> switch to -> new branch : 새로운 branch 생성
- 다른 branch로 변경하는 것을 check out 이라고 부름
- Git은 처음에는 master branch(main branch)에서 시작
